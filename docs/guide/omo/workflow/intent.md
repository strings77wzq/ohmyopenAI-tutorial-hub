# 意图分类

意图分类是 OMO 的第一道关卡，决定如何处理用户请求。

## 分类逻辑

```
用户请求
     ↓
[复杂度评估]
     ├─ 信息查询 → Librarian/Explore 直接处理
     ├─ 简单修改 → Hephaestus 直接处理
     └─ 复杂任务 → Sisyphus 分解处理
```

## 判断标准

| 特征 | 简单查询 | 复杂任务 |
|------|---------|---------|
| 涉及文件 | 单文件 | 多文件 |
| 步骤 | 一步 | 多步 |
| 依赖 | 无依赖 | 有依赖 |
| 不确定性 | 低 | 高 |

## 显式触发

用户也可以显式指定模式：

```bash
# 使用 ultrawork (自动判断)
ulw

# 使用 Prometheus (采访式规划)
# Tab 键切换到 Plan 模式，然后 /start-work
```
