# Agent 分类与能力

OMO 提供 11 个专用 Agent，每个 Agent 有明确的职责和擅长的模型类型。

## Agent 列表

| Agent | 擅长模型类型 | 职责 | 典型场景 |
|------|-------------|------|---------|
| **Sisyphus** | 推理型 | 主编排器 | 复杂任务统筹 |
| **Hephaestus** | 执行型 | 代码实现 | 写代码、修 bug |
| **Prometheus** | 推理型 | 规划师 | 采访式需求澄清 |
| **Atlas** | 均衡型 | Todo 编排 | 任务拆解与执行 |
| **Oracle** | 推理型 | 架构咨询 | 复杂决策建议 |
| **Librarian** | 搜索型 | 文档搜索 | 查找参考资料 |
| **Explore** | 搜索型 | 代码探索 | 快速定位代码 |
| **Metis** | 推理型 | 计划咨询 | 前期规划分析 |
| **Momus** | 快速型 | 计划审查 | 质量把关 |
| **Visual** | 视觉型 | UI/UX | 前端开发 |
| **General** | 均衡型 | 通用任务 | 兜底处理 |

## 模型选择原则

**不绑定特定模型**，只推荐模型类型：

| 任务类型 | 推荐模型特性 |
|----------|-------------|
| 架构设计 | 强推理、长上下文 |
| 代码执行 | 高效、准确 |
| 快速搜索 | 低延迟、低成本 |
| 视觉设计 | 图像理解能力强 |

## Agent 协作模式

### 串行模式
```
Sisyphus → Prometheus → Hephaestus → 验证
```

### 并行模式
```
Sisyphus → ┬─→ Agent A
           ├─→ Agent B
           └─→ Agent C → 汇总
```

```
