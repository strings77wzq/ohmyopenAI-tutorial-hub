# 主编排器设计

Sisyphus 是 OMO 的主编排器，负责：分解任务、分发子 Agent、收集结果、验证完成。

## 核心职责

```
1. 接收用户请求
2. 判断复杂度 → 意图分类
3. 任务分解 → 生成子任务列表
4. 分发到专用 Agent
5. 收集结果 → 汇总
6. 验证完成 → 或迭代修复
```

## 任务分解策略

| 复杂度 | 策略 | 示例 |
|--------|------|------|
| **简单** | 直接处理 | 单文件修改 |
| **中等** | 顺序执行 | A → B → C |
| **复杂** | 并行 + 汇总 | A,B,C 并行 → D 汇总 |

## 结果验证

Sisyphus 不只是分发任务，还负责验证：
- **自验证**：Agent 完成后自己检查结果
- **交叉验证**：另一个 Agent 审核
- **独立验证**：用 Hephaestus 验证 Hephaestus

## 与传统单 Agent 对比

| | 单 Agent (Claude Code) | Sisyphus (OMO) |
|---|---|---|
| 任务处理 | 单线程 | 多 Agent 并行 |
| 上下文 | 窗口耗尽 | 分工保持精简 |
| 验证 | 人工检查 | 自动验证回路 |
| 扩展性 | 受限于单模型 | 多模型协作 |
