# 检索与知识

检索让 Agent 不必把所有项目知识塞进 prompt，而是在需要时查找最相关的文档、代码和记录。

## 适合检索的内容

- API 文档和版本说明。
- 项目架构说明。
- 代码符号和调用关系。
- 历史决策、OpenSpec 变更和故障记录。
- 运行日志和测试失败摘要。

## 检索质量标准

| 标准 | 说明 |
| --- | --- |
| 相关性 | 结果直接支持当前问题 |
| 新鲜度 | 来源没有过期 |
| 可追踪 | 能回到文件、URL 或提交 |
| 小而准 | 摘要足够短，不淹没上下文 |
| 可验证 | 结论可以被测试或二次读取确认 |

## 与 MCP 的关系

MCP 可以把检索能力包装成 resource 或 tool。区别在于：

- resource 适合读取稳定资料。
- tool 适合带参数的搜索、过滤或聚合。

## 练习

为教程站设计一个检索策略：

1. 哪些目录应该被索引？
2. 检索结果最多返回几条？
3. 每条结果需要哪些字段？
4. 如何避免返回过期页面或 404 链接？
