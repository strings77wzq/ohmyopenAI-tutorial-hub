# Agent 工作流编排

稳定的 Agent 工作流不是“让模型一次性解决所有问题”，而是把意图、上下文、工具和验证拆成可恢复的步骤。

## 基础闭环

1. 明确目标：写下用户意图、范围和验收标准。
2. 读取上下文：优先读取本地项目文件和官方文档。
3. 制定计划：拆成可验证的小任务。
4. 执行变更：保持 diff 小，遵守项目约定。
5. 验证结果：构建、测试、链接检查、视觉检查。
6. 记录状态：更新任务清单和剩余风险。

## 如何连接本教程模块

| 模块 | 在工作流中的位置 |
| --- | --- |
| Skills | 封装可重复执行的操作 |
| MCP | 提供工具、资源和外部上下文 |
| OpenSpec | 记录需求、设计和任务 |
| Harness | 验证行为、失败模式和回归 |
| Evaluation | 决定是否达到发布标准 |

## 编排检查清单

- 每一步都有输入和输出。
- 工具调用有权限边界。
- 失败后能恢复或回滚。
- 关键行为有 Harness 或构建验证。
- 最终报告包含证据，不只描述“已完成”。
