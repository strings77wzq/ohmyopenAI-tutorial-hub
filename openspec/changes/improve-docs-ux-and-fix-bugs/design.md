# 设计方案：文档改进

## 设计原则

### 1. 操作优先原则
- 先告诉用户**怎么做**
- 再解释**为什么**
- 概念解释作为"延伸阅读"或"了解更多"

### 2. 渐进式披露
- 首页只展示最核心的入口
- 详情逐步展开
- 避免一次性给出过多选择

### 3. 一致性
- 所有命令首次出现时附加中文说明
- 代码块格式统一
- 术语使用一致

## 具体改进方案

### 改进 1：首页重构 (docs/index.md)

**当前问题**：
- 功能卡片在前，操作入口在后
- 角色选择造成决策负担

**改进方案**：
```
英雄区（Hero）
├── 标题：AI 编程教程
├── 副标题：系统学习 Skills、OpenSpec、Harness
├── 主要按钮：[5 分钟快速开始] ← 最显眼

快速预览（3 个卡片简化版）

### 改进 2：快速开始重构 (docs/guide/quickstart.md)

**当前结构**：
1. 理解核心概念（理论）
2. 创建你的第一个 Skill（操作）

**新结构**：
```markdown
# 快速开始

## 第一步：创建你的第一个 Skill（2 分钟）

直接给代码：
```json
{
  "name": "hello-world",
  "description": "简单的问候 Skill",
  "prompt": "回复一句问候语",
  "examples": [{"input": "你好", "output": "你好！很高兴见到你。"}]
}
```

保存到 `.skills/hello-world.json`

## 第二步：运行它（1 分钟）

在 AI 助手中输入：`/hello-world`

## 第三步：理解它（可选）

（这里才是概念解释）
```
