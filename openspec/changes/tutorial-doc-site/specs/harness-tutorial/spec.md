## ADDED Requirements

### Requirement: Harness 教程必须介绍测试基础设施概念
解释 Harness 是什么、为什么需要测试基础设施、Harness 在 AI 编程中的角色。

#### Scenario: 概念理解
- **WHEN** 初学者阅读 Harness 概念章节
- **THEN** 能解释 Harness 的作用：评估 AI 输出质量、确保工具调用正确性

### Requirement: Harness 教程必须包含编写测试场景的教学
教学如何编写测试场景（scenarios）、如何定义测试用例、如何组织测试目录。

#### Scenario: 测试编写
- **WHEN** 用户按照教程编写测试场景
- **THEN** 能创建可运行的测试用例，并理解测试结构

### Requirement: Harness 教程必须包含 Evaluators 使用指南
教学如何使用 Evaluators 评估 AI 输出质量，包括文本完整性、工具正确性等评估维度。

#### Scenario: 评估器使用
- **WHEN** 用户需要评估 AI 输出
- **THEN** 能配置和使用 Evaluators 进行自动化评估

### Requirement: Harness 教程必须包含 Mock Server 使用说明
教学如何使用 Mock Server 进行测试，避免产生实际 API 费用。

#### Scenario: Mock Server 运行
- **WHEN** 用户启动 Mock Server 运行测试
- **THEN** 测试能在不消耗 API 额度的情况下完成

### Requirement: Harness 教程必须包含完整实战案例
提供一个完整的测试用例编写实战，从场景定义到测试执行到结果分析。

#### Scenario: 实战案例跟随
- **WHEN** 用户跟随实战案例
- **THEN** 能完成一个完整的测试用例，理解 Harness 的完整工作流
