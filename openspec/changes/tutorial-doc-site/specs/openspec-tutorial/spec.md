## ADDED Requirements

### Requirement: OpenSpec 教程必须介绍核心概念
解释什么是 Spec-Driven Development（SDD）、OpenSpec 的核心哲学、与传统开发模式的区别。

#### Scenario: 概念理解
- **WHEN** 初学者阅读核心概念章节
- **THEN** 能解释 SDD 的工作流程：意图 → Spec → AI → 代码 & 验证

### Requirement: OpenSpec 教程必须包含完整工作流教学
教学 `/opsx:propose` → `/opsx:apply` → `/opsx:archive` 的完整工作流，每个命令提供使用示例和注意事项。

#### Scenario: 工作流跟随
- **WHEN** 用户按照工作流教程操作
- **THEN** 能完成一个完整的 OpenSpec 变更周期（提案 → 实现 → 归档）

### Requirement: OpenSpec 教程必须包含实战案例
使用电商场景作为实战案例，展示如何从 Spec 驱动代码实现，提供多语言实现对比。

#### Scenario: 实战案例复现
- **WHEN** 用户运行教程中的示例项目
- **THEN** 能理解 OpenSpec 如何驱动多语言实现

### Requirement: OpenSpec 教程必须包含最佳实践
总结编写高质量 Spec 的技巧、常见错误、与 AI 协作的模式。

#### Scenario: 最佳实践应用
- **WHEN** 用户编写自己的 Spec
- **THEN** 能应用教程中的最佳实践，避免常见陷阱

### Requirement: OpenSpec 教程必须包含命令参考表
提供所有 `/opsx:` 命令的参考表，包含命令说明、使用频率、示例。

#### Scenario: 命令查询
- **WHEN** 用户需要查找某个命令的用法
- **THEN** 能在参考表中快速找到命令说明和示例
