## ADDED Requirements

### Requirement: 学习路径必须提供角色化入口
首页必须提供三个角色化入口：Full-stack Developer、Architect、Student，每个角色有明确的学习路径。

#### Scenario: 角色选择
- **WHEN** 用户访问首页
- **THEN** 能看到三个角色卡片，点击后跳转到对应的学习路径

### Requirement: 学习路径必须具有渐进式复杂度
每个路径的内容组织必须遵循渐进式复杂度：概念入门 → 基础操作 → 进阶技巧 → 实战案例。

#### Scenario: 渐进学习
- **WHEN** 用户按照路径顺序学习
- **THEN** 难度逐步提升，不会出现知识跳跃

### Requirement: 学习路径必须标注难度级别
每个章节必须标注难度级别（入门/初级/中级/高级），帮助读者选择合适的起点。

#### Scenario: 难度筛选
- **WHEN** 用户查看目录
- **THEN** 能清晰看到每个章节的难度标注

### Requirement: 学习路径必须提供前置知识说明
每个章节开头必须说明所需前置知识，避免读者盲目进入。

#### Scenario: 前置检查
- **WHEN** 用户准备开始某章节
- **THEN** 能看到该章节需要的前置知识，判断是否准备好
