# 设计方案：修复显示异常

## 设计原则

1. **可靠性优先** - 使用文字而非 emoji，确保 100% 显示成功率
2. **原创性设计** - 避免标准 Markdown 表格，使用卡片式布局
3. **渐进增强** - 基础内容可访问，视觉效果作为增强

## 具体改进方案

### 1. Emoji 替换方案

| 原 Emoji | 替换方案 | 示例 |
|---------|---------|------|
| ✅ | (✓) 或 [正确] | (✓) 好的命名 |
| ❌ | (✗) 或 [错误] | (✗) 不好的命名 |
| 🔴 | 【高】或 [P0] | 【高】安全问题 |
| 🟡 | 【中】或 [P1] | 【中】建议改进 |
| 🟢 | 【低】或 [P2] | 【低】提示信息 |
| 🧠 | [技能] | [技能] 模块 |
| 📋 | [规范] | [规范] 模块 |
| 🧪 | [测试] | [测试] 模块 |
| 🚀 | [实战] | [实战] 项目 |
| ⚡ | [快速] | [快速] 开始 |
| ⭐ | 文字描述 | 使用频率：高 |

### 2. 表格重新设计

**原设计（标准表格）**:
```markdown
| 命令 | 说明 | 频率 |
|------|------|------|
| /opsx:propose | 创建提案 | ⭐⭐⭐⭐⭐ |
```

**新设计（卡片式）**:
```markdown
## 命令概览

### /opsx:propose（创建提案）
**使用频率**：必须 | **学习优先级**：高

创建新功能提案，AI 自动生成规范文档。

---

### /opsx:apply（应用规范）
**使用频率**：必须 | **学习优先级**：高

按规范实现代码，支持指定任务或阶段。
```

### 3. 首页重新设计

**原设计**:
```markdown
features:
  - icon: 🧠
    title: Skills
```

**新设计**:
```markdown
features:
  - icon: /icons/skill.svg
    title: Skills
    titleText: 技能系统
```

或使用纯文字标签：
```markdown
## 核心模块

**[技能系统]** 学习如何创建可复用的 AI 技能...
**[规范驱动]** 掌握 OpenSpec 规范驱动开发...
**[测试保障]** 构建 Harness 测试基础设施...
```

### 4. 优先级标记重新设计

**原设计**:
```markdown
| 命令 | 优先级 |
|------|--------|
| /opsx:propose | ⭐⭐⭐⭐⭐ |
```

**新设计**:
```markdown
**优先级分类**：
- 【必须】核心命令，学习优先级最高
- 【重要】常用命令，建议掌握
- 【可选】进阶功能，按需学习
```

## 实施计划

1. 替换所有 emoji 为文字标记
2. 重新设计表格为卡片式布局
3. 更新首页 features 配置
4. 更新所有文档中的图标引用
5. 本地构建测试
6. 提交和推送

## CSS 增强（可选）

如需视觉效果，可在 VitePress 配置中添加自定义 CSS：

```css
.priority-high { color: #dc2626; font-weight: bold; }
.priority-medium { color: #d97706; }
.priority-low { color: #059669; }
```
