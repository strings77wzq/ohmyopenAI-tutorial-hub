# 改进文档 UX 并修复显示 Bug

## 问题概述

当前文档站点存在以下核心问题：

### 1. 界面不直观
- 首页功能展示优先，缺少即时操作入口
- 角色选择增加用户认知负担
- 用户不知道该从哪里开始

### 2. 结构问题：原理先于操作
- "快速开始"页面先讲概念再讲操作，违背直觉
- 学习路径中穿插概念理解和动手实践，混淆不清

### 3. 显示 Bug
- 代码块缺少标记导致渲染异常
- 格式不一致

### 4. 术语使用
- 部分英文命令缺少中文说明
- 首次出现时未解释

## 目标

1. **首页重构**：让用户 5 秒内知道如何开始
2. **快速开始重构**：先给操作指令，再讲原理
3. **修复显示 Bug**：修正所有格式问题
4. **术语标准化**：命令首次出现时附加中文说明

## 参考标准

对比优秀开源项目（Vue、React、VitePress）：
- ✅ 首页立即展示"Get Started"按钮
- ✅ 快速开始页面首屏就是代码示例
- ✅ 概念解释放在操作之后作为"延伸阅读"
- ✅ 所有命令都有清晰的步骤说明

## 预期结果

- 新用户访问首页后立即知道点击哪里
- "快速开始"页面能让用户在 5 分钟内完成第一个 Skill
- 所有页面无显示 Bug
- 术语一致且易懂
